ClickUp как интерфейс (AI Director)

Зачем ClickUp вместо Telegram

Telegram ClickUp
Визуальный статус Нет Kanban board — видно все задачи
Команда 1 человек Вся команда видит прогресс
Возврат на доработку Написать "переделай" Перетащить карточку назад
История Скролить чат Всё в карточке задачи
Файлы В чате (теряются) В задаче (прикреплены)

Board структура

📥 Reference    → видео-референс кинули сюда
🔍 Analysis     → AI проанализировал, написал план в комментарии
👤 Avatar       → выбран аватар
✅ Approved     → план одобрен
🎨 Frames      → NanoBanana генерит кадры (прикреплены к задаче)
🎬 Video       → Kling/Vidu генерит видео
✂️ Assembly    → склейка + музыка + субтитры
👀 Review      → финальное видео на ревью
📱 Post        → одобрено, ждёт постинга
✅ Posted      → опубликовано, собираем метрики

Как работает технически

1. Человек создаёт задачу в "Reference" → прикрепляет видео
2. ClickUp webhook → сервер ловит → Claude API анализирует
3. Claude пишет план в комментарий к задаче → двигает в "Analysis"
4. Человек читает план → одобряет → двигает в "Approved"
5. Webhook ловит → pipeline генерит → прикрепляет кадры → "Frames"
6. Кадры одобрены → генерит видео → "Video" → "Assembly"
7. Финальное видео прикреплено → "Review"
8. Одобрено → "Post" → автопостинг или ручной

ClickUp API — возможности и ограничения

Что МОЖНО: - Создавать/обновлять задачи - Писать комментарии - Двигать по статусам - Прикреплять файлы (attachments) - Ловить webhooks (задача перемещена, комментарий добавлен) - Кастомные поля (аватар, hook, pipeline type, cost)

Ограничения: - Файлы: максимум ~100MB на attachment (видео 720p обычно 2-10MB — ок) - Rate limit: 100 запросов в минуту (достаточно) - Webhooks: нужен публичный URL (сервер с доменом)

Для нашего flow проблем нет — видео 720p весят 1-10MB, это далеко от лимита.
